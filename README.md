# cn-kai-journal

本仓库用于归档和发布多个独立 HTML 页面，内容涵盖各类资料整理与信息展示。仓库本身不针对任何特定域名或网站，所有页面均为静态 HTML 文件，可直接通过浏览器打开查看。

## 项目简介

这是一个以 HTML 页面为主要载体的归档仓库，旨在以简单、开放的方式保存和分享各类文档、笔记、索引页等静态内容。页面之间相互独立，便于引用、分发和本地浏览。

## 目录说明

```
cn-kai-journal/
├── index.html          # 仓库首页，可包含页面导航
├── pages/              # 存放各个独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/             # 页面所需的资源文件（如图片、样式、脚本）
    ├── css/
    ├── js/
    └── images/
```

- **index.html**：仓库入口页面，可列出所有归档页面的链接或简要描述。
- **pages/**：每个 HTML 页面代表一个独立归档单元，命名应简洁明了。
- **assets/**：存放页面共用的或单独引用的静态资源，建议按类型划分子目录。

## 页面归档说明

- 每个 HTML 页面均使用标准 HTML5 语法，不依赖外部复杂框架。
- 页面内容以文本、表格、图片、链接等形式呈现，确保在主流浏览器中正常显示。
- 归档页面不包含任何动态后台功能，所有数据均内嵌或通过静态资源引用。
- 页面之间无硬性依赖，可单独复制使用。

## 维护说明

- **新增页面**：在 `pages/` 目录下创建新 HTML 文件，并视需要在 `index.html` 中添加导航链接。
- **更新页面**：直接编辑对应 HTML 文件，更新内容后提交即可。
- **资源管理**：新增或修改资源文件时，请保持目录结构清晰，避免文件散落。
- **提交规范**：提交信息应简要说明变更内容，如“新增 XX 页面”或“更新 XX 页面格式”。
